Transaction-based Indices

LO 15.8: Demonstrate knowledge of transaction-based real estate indices.

For example:

  • Identify the key characteristics of transaction-based real estate indices.
  • Discuss the repeat-sales method (RSM) for estimating transaction-based price indices, including two key advantages and three key disadvantages, and apply the process for using the RSM to construct an index.
  • Discuss the hedonic pricing method (HPM) for estimating transaction-based price indices, including its five key advantages and six key disadvantages, and apply the three-step process for calculating a hedonic price index.
  • Compare and contrast RSM and HPM indices.
  • Describe sample selection biases associated with transaction-based indices.

While appraisal-based indices partly rely on transaction data, a transaction-based index is based wholly on reported sale prices in real estate transactions. If the index is accurate, derivatives can be based on the index. To ensure accuracy, the index must have sufficient data, adjust for property characteristics, and have minimal unexplained variation. The primary methods used to construct transaction-based indices are the repeat-sales method and the hedonic pricing method.

The repeat-sales method (RSM):

  • It is based on changes in the values of properties over an observation period based on at least two reported transactions.
  • The returns of properties that have changed hands two or more times are used to infer price changes for all properties.
  • The observed percentage price changes are regressed onto time-dummy variables. For each property (i.e., observation), the dependent variable is the log of the holding-period return, while time-dummy variables are the independent variables for each time period and will have a value of 1 if the property was held during that time period and 0 if it was not.

Advantages of RSM include:

  • RSM uses sales prices of the same property and, therefore, does not have a problem with differences in property attributes affecting the prices. RSM does not need detailed property characteristics.
  • RSM is relatively robust to specification error (i.e., choice of wrong explanatory variables in a regression model).

Drawbacks of RSM include:

  • RSM captures value information representing only a small sample of properties with multiple sales during the measurement window. Values of properties that sold only once during the sample period may represent valuable data but are ignored under RSM.
  • Prices may reflect more than returns. For example, a property might be purchased, undergo major improvements, and be sold again at a much higher price. These idiosyncratic changes in value are reflected in the index but are not necessarily representative of true market gain.
  • Updates to RSM results in “backward” adjustments. This is so because when a property is sold, the holding period return is allocated to all the periods intervening since the property was previously bought (i.e., the returns are backfilled).

Indices that utilize the hedonic pricing method (HPM) use observed transaction data of some properties and the characteristics of other, dissimilar properties, to estimate prices of all properties, including properties that did not change hands. For example, assume one retail space is sold. That information can be used to value other retail spaces that are dissimilar in terms of location, square footage, parking, quality of anchor stores, and so on. The value of a real estate property can be thought of as sum total of the value of all the attributes that the property possesses. If we can discretely value each of the attributes, we can value other properties that have some but not all of the same attributes.

The steps for creating an index using HPM are:

  1. Specify characteristics of properties that are related to value. In the above example of retail properties, that is location, square footage, parking, and quality of anchor stores.
  2. Use prices of recent transactions to fit the parameters of the valuation model.
  3. Use the estimated model parameters to value other, nontransacted properties.

Professor’s Note: The primary difference between the two methods is that the hedonic indices include all properties that have sold at least once, while RSM indices include data only on properties that have sold multiple times. Also, hedonic price indices use price levels on some properties to infer price levels on other properties. RSM uses price changes on some properties to infer price changes on other real estate. The accuracy of the pricing model is critical to the success of either of these approaches.

Example: Hedonic pricing model

A regression model for valuing retail real estate properties is as follows:

ln(value) = 3.457 + (0.022 × size) – (1.2268 × age) + (0.16 × sales)

where:

price = value of the property

size = square footage of the property in thousands of sq. feet

age = age in years

sales = revenue in millions of dollars

Estimate the value of a 23-year-old, 300,000-square-foot retail complex with annual sales of $230 million.

Answer:

ln(value) = 3.457 + (0.022 × 300) – (1.2268 × 23) + (0.16 × 230) = 18.64

value = e18.64 = $124,597,612

Advantages of HPM:

  • HPM uses all transaction data (not just repeat sales)—especially important when sample size is small.
  • HPM is versatile and can account for important internal and external property attributes (e.g., quality, size, air pollution).
  • HPM allows for analyzing the contribution of individual attributes in value determination that may be otherwise difficult to value (e.g., school quality, air pollution).
  • Marginal contribution of individual attribute can be determined using HPM.
  • Unlike RSM, no backward adjustment is necessary under HPM.

Disadvantages of HPM:

  • Requires large sample size, which may be costly.
  • Similar to RSM, HPM suffers from sample selection bias—model estimates are only based on actual transactions.
  • Potential for specification error (all important attributes may not be included in the model).
  • Assumes that parties to the transaction not only agree on the same attributes as value determinants but also on the value of the said attributes.
  • Oftentimes, HPM may suffer from multicollinearity (i.e., when the specified attributes are highly correlated). For example, it may be difficult to identify how attributes in a new property development (e.g., low crime, location, size, quality) influence the price if comparable developments with only one variable change (e.g., location) are unavailable.
  • HPM assumes that a prospective buyer can easily find a property with the specified attribute levels at a given price. In reality, this is not possible.

RSM vs. HPM

Both models suffer from sample selection bias (i.e., fail to represent the entire population of properties). RSM includes only those properties that have multiple sales, while HPM includes all properties that have sold at least once. As a result, HPM has less sample-selection bias due to its larger sample size.

The NCREIF Transaction Based Index (TBI) utilizes the hedonic pricing model, but also has similarities to the repeat sales method. It uses both the appraisal values from the NCREIF Property Index (NPI) and transaction prices from NPI properties. The TBI is published quarterly.

Sample-Selection Bias

The sample for transaction-based indices includes only those properties that have actually sold. This may result in a biased sample. Suppose that sellers are more likely to sell those properties that have appreciated in real terms as opposed to those that have remained flat or even declined. The sample of properties that have actually sold is now more likely to include appreciated properties—a biased sample.
