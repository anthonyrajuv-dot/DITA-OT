The above example shows it is possible to unsmooth a reported return series. Returns must follow a first-order autocorrelation process without an error term. Also,  must be estimated without error. In practice, unsmoothed data contains errors. It is unlikely that  can be estimated without error and that all return series will follow a first-order autocorrelation process perfectly.

There are three basic steps to unsmoothing data.

Step 1: Specify the form of autocorrelation (i.e., first-order or higher orders).

Step 2: Estimate the parameters of the assumed autocorrelation process (i.e., first order or higher orders). This is often the first-order autocorrelation parameter, . The autocorrelation parameter is the correlation coefficient between each observation and the proceeding observation in the series. Small sample sizes can lead to large estimation errors. Step 3: Insert the estimated correlation coefficient () in the true return equation we previously covered and solve for the true but unobservable return.

For example, in the previous example, the true correlation coefficient is 0.7. If the analyst estimates  = 0.4 instead of 0.7, she can re-estimate the true returns using the above equation by using the 0.4 estimate of . However, because the estimated  is not the true correlation, there will be errors in estimating the true returns. For example, estimating the true return in Period 3 using the estimated  yields R3,true = [2.4% – (0.4 × 0%)] /(1 – 0.4) = 4%. This is half of the 8% true return estimated and the true correlation of 0.7.
